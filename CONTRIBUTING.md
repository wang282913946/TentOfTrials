# Contributing to TentOfTrials

Thank you for your interest in contributing to TentOfTrials! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: git clone https://github.com/YOUR_USERNAME/TentOfTrials.git
3. Install dependencies: pip install -r requirements.txt
4. Create a feature branch: git checkout -b feature/your-feature-name

## Building the Project

`ash
# Run the build script
python3 build.py

# Run with module selection
python3 build.py --modules core,utils

# Generate build diagnostics
python3 build.py --diagnostics
`

## Development Workflow

1. **Create an issue** for bugs or features
2. **Discuss** your approach in the issue comments
3. **Implement** your changes
4. **Add tests** for new functionality
5. **Update documentation** as needed
6. **Submit a pull request**

## Pull Request Guidelines

- Follow the PR template
- Include a clear description of changes
- Reference related issues
- Ensure all tests pass
- Keep PRs focused and atomic

## Coding Standards

- Follow PEP 8 for Python code
- Use type hints where applicable
- Write clear, descriptive commit messages
- Document public APIs

## Bounty Program

This project offers bounties for specific tasks. Check the [bounty issues](../../issues?q=label%3Abounty) for available tasks.

## Questions?

Feel free to open an issue for questions or discussion.
